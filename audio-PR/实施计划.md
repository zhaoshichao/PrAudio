# Premiere Pro 音频插件面板 — 实施计划

## 背景

开发一个 CEP (Common Extensibility Platform) 面板插件，用于在 Premiere Pro 中浏览、预览、导入分类音频文件，并快速将其插入时间轴音轨。

## 目标

- 左侧：音乐分类列表（流行音乐/怀旧音乐/经典音乐 × 30秒/60秒/90秒）
- 右侧：对应音频文件列表，显示文件名、导入状态、预览按钮、导入按钮
- 已导入的音频可直接双击或点击按钮插入到当前播放头位置的音轨

## 音频文件来源策略（混合模式）

采用**方法 2（混合模式）**：以文件夹结构扫描为主，JSON 配置提供覆盖/排除/补充能力。架构预留扩展点，未来可接入网络/数据库源（方法 3）。

### 发现流程

```
discoverAudioFiles(categoryKey, subKey)
  │
  ├─ 1. 读取 categories.json 获取该子类的配置
  │
  ├─ 2. 文件夹扫描（主数据源）
  │    扫描 audio_files/{category}/{sub}/*.mp3, *.wav, *.aiff, *.flac
  │    每个文件自动生成条目：{fileName, filePath, duration: null}
  │
  ├─ 3. JSON 覆盖层（修正/补充）
  │    - exclude: 排除不想要的音频文件（如临时文件、草稿）
  │    - files[].fileName: 为特定文件补充元数据（自定义显示名、时长标签）
  │    - files[].filePath: 额外添加不在文件夹中的文件（如外部路径引用）
  │    - order: 控制排序（default / alphabetical / custom）
  │
  └─ 4. 合并结果 → 返回统一的 AudioFile[] 数组
```

### 未来扩展至方法 3 的设计

数据获取逻辑封装在 `getAudioFilesForCategory(categoryKey, subKey)` 函数中，该函数返回统一的 `AudioFile[]` 数组。未来只需：

1. 在 `categories.json` 中为子类添加 `source: "network"` 字段
2. 实现一个新的 source handler（如 `fetchFromNetwork()`）
3. 在 `getAudioFilesForCategory` 中根据 `source` 字段分发

面板 UI 层完全不感知数据来源差异。

## 技术架构

### 通信模式

```
panel.js (CEF/HTML5)  ←→  CSInterface.evalScript()  ←→  audio_backend.jsx (ExtendScript)
                              app.setSDKEventMessage()
```

- Panel→ExtendScript: `csInterface.evalScript('$._AUDIO_PLUGIN_.func(args)')`
- ExtendScript→Panel: `app.setSDKEventMessage(JSON.stringify(msg))` → SDK event listener

### 关键设计决策

1. **不依赖外部框架**：纯 HTML/CSS/JS，零依赖，直接可运行
2. **混合音频源**：文件夹扫描为主，JSON 提供覆盖/排除/补充，架构预留网络源扩展点
3. **文件 I/O 走 ExtendScript**：CEF 沙箱无法直接读本地文件，所有文件操作通过 ExtendScript
4. **拖拽替代方案**：CEP 面板与 Premiere Pro 之间的跨进程 HTML5 拖拽不可行。采用**双击/按钮插入到播放头位置**作为主交互方式
5. **导入状态追踪**：ExtendScript 端维护 `importedFileMap`（路径→项目项引用），初始化时扫描项目重建

### 核心 ExtendScript 函数（`$._AUDIO_PLUGIN_`）

| 函数 | 功能 |
|------|------|
| `loadConfig()` | 读取 categories.json，解析 JSON 配置 |
| `discoverAudioFiles(categoryKey, subKey)` | 扫描文件夹 + 应用 JSON 覆盖 → 返回统一 AudioFile[] |
| `previewAudio(filePath)` | 在 Source Monitor 中打开并播放音频（抑制自动波形生成） |
| `stopPreview()` | 关闭 Source Monitor 预览 |
| `importAudio(filePath)` | 导入单个音频文件到项目的 "Audio Plugin Imports" Bin |
| `insertAudioAtPlayhead(filePath)` | 将已导入音频插入到播放头位置的第一音轨 |
| `scanProjectForImportedAudio()` | 扫描项目重建导入状态映射 |

### discoverAudioFiles 合并逻辑（关键）

```
discoverAudioFiles(categoryKey, subKey):
  1. 从 categories.json 获取 subConfig
  2. folderScanResult = scanFolder(audioBasePath + "/" + categoryLabel + "/" + subConfig.folderName)
     → [{fileName, filePath, absolutePath}]
  3. 应用 exclude 模式：过滤 folderScanResult 中匹配 exclude 的文件
  4. 合并 JSON files[]：添加 filePath 不在 folderScanResult 中的条目（外部引用）
  5. 应用 JSON files[] 覆盖：替换同名文件的 displayName、durationLabel 等
  6. 应用 order 排序
  7. 交叉查询 importedFileMap，填充每个文件的导入状态
  8. 返回 AudioFile[] = [{fileName, filePath, displayName, durationLabel, isImported, projectItemId}]
```

### 复用现有代码（来自 `4 常用代码_Premiere.jsx`）

- `$._PPP_.updateEventPanel(message)` → `app.setSDKEventMessage(message, 'info')` — ExtendScript→Panel 消息发送
- `$._PPP_.searchForBinWithName(name)` (line 420) — 递归搜索项目 Bin
- `$._PPP_.openInSource()` (line 373) — Source Monitor 预览模式（含 AE 波形生成抑制技巧）
- `app.project.importFiles(paths, suppressWarnings, targetBin, importAsStills)` (line 457) — 文件导入
- `track.insertClip(projectItem, timeInSeconds)` (line 1669) — 剪辑插入轨道
- `track.overwriteClip(projectItem, timeInSeconds)` (line 1710) — 剪辑覆盖到轨道
- `app.sourceMonitor.closeClip()` (line 1723) — 关闭 Source 预览
- `CSXSEvent` 模式 (line 1398) — 发送 SDK 事件到面板

## 实施步骤

### Step 1: 创建目录结构和配置文件
- 创建 `Audio/CSXS/`, `Audio/host/`, `Audio/jsx/`, `Audio/config/`, `Audio/audio_files/` 目录
- 写入 `config/categories.json`（含混合模式配置、exclude/files 覆盖字段）
- 创建示例音频文件目录结构
- 创建 `.debug` 空文件

### Step 2: 编写 CEP 清单
- `CSXS/manifest.xml` — 注册扩展，绑定 Host=PPRO，主面板指向 `host/index.html`，脚本指向 `jsx/audio_backend.jsx`

### Step 3: 编写 ExtendScript 后端
- `jsx/audio_backend.jsx` — 实现 `$._AUDIO_PLUGIN_` 命名空间
  - `loadConfig()`: 读取并解析 categories.json
  - `discoverAudioFiles(categoryKey, subKey)`: 文件夹扫描 + JSON 覆盖合并逻辑
  - `previewAudio(filePath)`: Source Monitor 预览（复用 openInSource 模式）
  - `stopPreview()`: 关闭预览
  - `importAudio(filePath)`: 导入到 "Audio Plugin Imports" Bin
  - `insertAudioAtPlayhead(filePath)`: 获取播放头位置 → 插入到第一个音轨
  - `scanProjectForImportedAudio()`: 遍历项目项重建 importedFileMap
  - Source adapter 结构：`getAudioFilesForCategory()` 作为数据获取入口，便于未来切换源
- 使用 `$.evalFile()` 引入 `4 常用代码_Premiere.jsx` 中的工具函数

### Step 4: 编写面板 HTML + CSS
- `host/index.html` — 左右双栏布局骨架，顶部搜索栏
- `host/style.css` — 暗色主题，分类树 + 音频列表样式，导入/预览/插入状态按钮

### Step 5: 编写面板 JS 主逻辑
- `host/panel.js` — CSInterface 初始化、配置加载、分类树渲染、音频列表渲染、过滤搜索、预览/导入/插入事件处理
- 面板 JS 只处理 UI 渲染和用户交互，数据获取通过 `evalScript` 调 ExtendScript

### Step 6: 端到端联调
- 将 `Audio/` 目录部署（symlink 或复制）到 CEP extensions 目录
- 在 Premiere Pro 中加载面板，测试完整流程

## 部署配置

```bash
# macOS CEP 扩展目录
~/Library/Application Support/Adobe/CEP/extensions/

# 启用 PlayerDebugMode（允许加载未签名扩展）
defaults write com.adobe.CSXS.6 PlayerDebugMode 1

# 创建符号链接进行开发
ln -sf /path/to/Audio ~/Library/Application\ Support/Adobe/CEP/extensions/Audio
```

## 验证方式

1. 在 Premiere Pro 中打开面板（窗口 → 扩展 → Audio Plugin）
2. 点击左侧分类子类 → 右侧显示对应音频文件列表（来自文件夹扫描 + JSON 合并）
3. 点击预览按钮 → Source Monitor 打开并播放音频
4. 点击导入按钮 → 文件导入项目 "Audio Plugin Imports" Bin，状态变为"已导入"
5. 双击已导入音频 / 点击插入按钮 → 音频插入到播放头位置的第一音轨
6. 搜索框输入关键字 → 音频列表实时过滤
7. 修改 categories.json 添加 exclude 或额外 files → 刷新后面板反映变更
